## 一、项目背景

### 1.1 社团活动形态

我们是一个大学内的**商业模式教学社团**，主要教学和活动内容包括两大模块：

| 模块 | 内容 | 形式 |
|------|------|------|
| **讲座模块** | 邀请嘉宾和教授开展讲座 | 单向输出，被动学习 |
| **小组研究模块** | 学员分组完成课题研究、讨论和展示 | 主动实践，协作学习 |

本产品聚焦于**小组研究模块**的AI赋能。

### 1.2 小组研究的标准流程

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  课题   │ →  │  小组   │ →  │  多次   │ →  │  成果   │ →  │  汇报   │
│  确定   │    │  组建   │    │  讨论   │    │  整理   │    │  展示   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  课题背景        成员背景        会议纪要       文档积累        演讲稿
  研究目标        专业能力        讨论要点       研究报告        PPT
  初步要求        个人偏好        分工安排       产品方案        展示效果
```

**小组规模**：7-10人/组

## 二、问题分析

### 2.1 现实痛点

在小组讨论场景中，存在**两类典型的参与者失衡**：

#### 类型一：高参与 / 低能力
- **表现**：表达积极、发言频繁
- **问题**：逻辑弱、观点浅，容易主导但质量不高
- **影响**：占据讨论空间，影响其他成员发挥

#### 类型二：低参与 / 高能力（偏I型）
- **表现**：有思考深度，但不愿发言或表达不充分
- **问题**：能力未被充分利用
- **影响**：优质观点流失，整体讨论质量下降

### 2.2 根本矛盾

```
┌─────────────────────────────────────────────────────────────┐
│                    当前状态 vs 期望状态                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   当前：讨论效率低，输出质量不稳定                            │
│        ↓                                                    │
│   问题：能力强的人没有被充分利用                              │
│        问题：能力弱的人没有被有效提升                          │
│                                                             │
│   期望：让每个人在最合适的位置参与，并产生有效输出              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 现有方案的局限

| 方案 | 局限 |
|------|------|
| 教师观察+点评 | 事后反馈，实时性差 |
| 小组自评 | 主观性强，深度不足 |
| 固定分工 | 缺乏灵活性，执行成本高 |

## 三、产品定位

### 3.1 核心定位

**不是"让大家多说"，而是：让每个人在最合适的位置参与，并产生有效输出。**

我们不是在优化讨论氛围，而是在用AI把"小组讨论"变成一个：
- **可控**的结构化训练场
- **可训练**的能力提升路径
- **可量化**的成长追踪体系

### 3.2 AI介入的价值主张

| 传统模式 | AI赋能模式 |
|----------|------------|
| 随机发言，无序讨论 | 结构化引导，角色分配 |
| 观点散落，缺乏整合 | 实时整理，要点提炼 |
| 事后总结，遗忘率高 | 持续记忆，智能累积 |
| 能力评估主观 | 多维度数据，客观分析 |
| 个人成长模糊 | 成长轨迹可视化 |

### 3.3 目标用户

| 用户角色 | 核心诉求 |
|----------|----------|
| 教师 | 全局掌控、进度跟踪、成果验收、评估依据 |
| 组长 | 高效组织、任务分配、进度推进、成果质量 |
| 组员 | 个人成长、轻松参与、获得反馈、能力提升 |

### 5.2 核心能力模块 (AI Skills)

每个Skill是Agent的一个**专业能力域**，通过特定的Prompt和工具实现：

| Skill | 功能描述 | 触发关键词 | 典型输出 |
|-------|----------|------------|----------|
| **调研助手** | 提供课题相关的市场、行业、竞品信息 | "调研"、"了解一下"、"市场分析" | 行业概览、竞品对比、市场数据 |
| **产品设计** | 协助设计产品/服务方案 | "设计产品"、"怎么设计"、"方案" | 产品方案文档、商业模式画布 |
| **分工规划** | 基于成员特点生成任务分配 | "分一下工"、"怎么分工"、"任务分配" | 任务分解表、责任人、时间节点 |
| **进度追踪** | 定期回顾进度，识别风险点 | "进度怎么样"、"看看进度"、"还剩什么" | 进度报告、风险提示、待办清单 |
| **汇报准备** | 协助准备展示PPT大纲、演讲稿 | "准备汇报"、"PPT"、"演讲稿" | 汇报大纲、PPT结构、演讲提示 |
| **会议主持** | 引导高效的小组讨论 | "开会"、"讨论一下"、"会议" | 议程、会议记录、会议总结 |

       │
       ▼

       │
       ▼

### 5.4 角色分配与发言辅助模块

#### 5.4.1 角色分配机制

根据用户特征自动分配讨论角色，避免"谁声音大谁主导"的无序状态：

| 角色 | 职责 | 适合特点 |
|------|------|----------|
| **分析者** | 拆解问题、评估方案、逻辑推演 | 逻辑思维强、结构化表达 |
| **表达者** | 总结归纳、汇报展示、观点整合 | 口头表达好、善于归纳 |
| **调研者** | 信息搜集、数据支撑、案例查找 | 信息检索能力强、数据敏感 |
| **质疑者** | 提出反对意见、发现漏洞、完善方案 | 批判性思维、追求严谨 |
| **推进者** | 控制节奏、引导方向、总结决议 | 组织能力强、善于协调 |

**动态调整**：
- 根据讨论内容动态调整角色
- 同一成员可在不同议题中承担不同角色
- 记录各角色表现，作为成长数据

#### 5.4.2 发言质量评估

**评估维度**：

| 维度 | 指标 | 反馈示例 |
|------|------|----------|
| 结构化 | 是否使用"结论-论据-案例"结构 | `你的发言结构很好，建议增加数据支撑` |
| 逻辑性 | 论点与结论是否一致 | `你的论据和结论之间还需要加强关联` |
| 数据支撑 | 是否有具体数据 | `建议补充具体数字或比例` |
| 案例引用 | 是否有具体案例 | `可以参考XX公司的做法` |
| 商业视角 | 是否从商业角度思考 | `从用户价值角度看，是否还有优化空间？` |

**实时反馈模式**：
```
成员发言 → AI分析 → 质量评分 → 改进建议（可选显示）
```
